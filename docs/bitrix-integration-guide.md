
# Руководство: Интеграция Bootstrap и React в 1С-Битрикс

Этот документ описывает общие принципы кастомизации платформы 1С-Битрикс с использованием современных фронтенд-технологий, таких как Bootstrap и React, для создания адаптивных, быстрых и интерактивных интерфейсов. Основной подход — перенос верстки в кастомные шаблоны компонентов Bitrix, подключение Bootstrap для стилизации, и создание API для взаимодействия с React-приложениями.

---

## Кастомизация с Bootstrap

Bootstrap де-факто является стандартом для реализации адаптивной верстки в проектах на Битрикс.

#### 1. Подключение
Файлы Bootstrap (CSS и JS) подключаются в заголовок и подвал шаблона сайта, которые находятся в файлах `header.php` и `footer.php` соответственно.

#### 2. Адаптация компонентов
- **Путь для кастомизации**: `/local/templates/ВАШ_ШАБЛОН/components/bitrix/`.
- **Процесс**: В эту директорию копируются стандартные компоненты из пространства `bitrix` (например, `bitrix:news.list` или `bitrix:menu`). После копирования HTML-разметка внутри шаблона компонента (`template.php`) изменяется для соответствия классам и структуре Bootstrap (использование сеток `.row > .col`, кнопок `.btn`, меню `.navbar` и т.д.).

#### 3. Интеграция из ядра (D7)
Современное ядро D7 позволяет использовать встроенные возможности для подключения стилей. Например, через вызов `$APPLICATION->SetAdditionalCSS()` в коде компонента или шаблона.

---

## Интеграция ReactJS

React используется для создания сложных интерактивных интерфейсов или полноценных одностраничных приложений (SPA - Single Page Application) внутри Битрикс.

#### Подход 1: Гибридный
- **Суть**: Компоненты React встраиваются в определенные места на стандартных, сервер-рендеренных страницах Битрикс.
- **Получение данных**: React-компонент после загрузки страницы получает необходимые ему данные от сервера через AJAX-запросы к специальным обработчикам в Битрикс.

#### Подход 2: API-driven (Headless)
- **Суть**: Битрикс выступает исключительно в роли бэкенда. Его задача — предоставлять данные в формате JSON через REST API или свои внутренние AJAX-обработчики.
- **Фронтенд**: React-приложение является полностью независимым фронтендом, который отвечает за всю логику отображения. Оно обращается к API Битрикса для получения, создания или обновления данных.

#### Сборка и подключение
React-приложение разрабатывается отдельно, а затем собирается с помощью сборщика (Webpack, Vite) в один или несколько статических `.js` и `.css` файлов. Эти собранные файлы подключаются в основном шаблоне сайта Битрикс (`header.php` или `footer.php`).

---

## Ключевые этапы работ

1.  **Копирование шаблона компонента**: **Правило №1** — никогда не изменять стандартные компоненты напрямую. Всегда работать с их копиями, размещенными в `/local/templates/`.
2.  **Создание AJAX-обработчиков**: Необходимо создавать PHP-файлы, которые будут принимать запросы от React, взаимодействовать с базой данных Битрикс и возвращать ответ в формате JSON.
3.  **Использование API D7**: При разработке обработчиков рекомендуется применять современные методы API ядра D7, в частности ORM (Object-Relational Mapping), для безопасной и эффективной выборки данных из базы.

Этот подход позволяет эффективно сочетать мощную и гибкую систему управления контентом, которой является Битрикс, и современный, отзывчивый пользовательский опыт, предоставляемый React и Bootstrap.
