2. Совместимость ShadCN UI и React 19
Поскольку Next.js 15 базируется на React 19, многие библиотеки компонентов сталкиваются с конфликтами зависимостей (peer dependencies).

Проблема: При установке ShadCN UI через npx shadcn@latest init часто возникают ошибки зависимостей, так как Radix UI или lucide-react могут требовать React 18.
Решение:

Для полной поддержки Tailwind v4 (если вы обновляетесь и на него) может потребоваться использование npx shadcn add canary.

3. Конфликты Tailwind CSS (v3 vs v4)
Next.js 15 часто идет в паре с обсуждением Tailwind CSS v4, который радикально меняет способ импорта стилей.

Проблема: В Tailwind v4 больше не используются директивы @tailwind base;, @tailwind components; и @tailwind utilities;. Вместо них используется один импорт @import "tailwindcss";.
Ошибки ShadCN: Инициализатор ShadCN может ожидать файл tailwind.config.js, в то время как в новых проектах часто используется tailwind.config.ts или CSS-конфигурация v4.
Совет: Если вы столкнулись с проблемами стилей, проверьте, не дублируются ли старые директивы @tailwind с новыми импортами в globals.css.

4. Изменения в TypeScript
Конфигурация: В tsconfig.json теперь рекомендуется использовать jsx: "react-jsx" и target: "ES2020" или выше для корректной работы с React 19.
Типизация роутов: Поскольку params стали асинхронными, типы для Page и Layout теперь должны отражать это (например, Promise<{ id: string }> для параметров).

5. Кэширование по умолчанию
В Next.js 15 кэширование запросов fetch, обработчиков GET и навигации на клиенте по умолчанию отключено (uncached by default). Это может привести к неожиданному увеличению нагрузки на сервер или API, если вы полагались на автоматическое кэширование из версии 14.

Практические шаги по миграции:
Обновите ядро: pnpm add next@latest react@latest react-dom@latest.
Запустите кодмод для асинхронных параметров: npx @next/codemod@canary next-async-request-api ..
Если используете ShadCN, обновите его компоненты вручную или переустановите с флагом --force, чтобы разрешить конфликты с React 19.
