# Современный стек для разработки с ИИ-агентами

Этот список инструментов представляет собой современный стек для разработки ПО с использованием ИИ-агентов. Основное различие между ними заключается в их роли: одни являются редакторами (где пишут код), другие — интеллектом (моделями), а третьи — связующим звеном.

## 1. Среды разработки и редакторы (IDE)

*   **Cursor:** Популярный ИИ-редактор, построенный на базе VS Code. Он глубоко интегрирует ИИ в рабочий процесс, позволяя редактировать целые файлы, исправлять ошибки и генерировать архитектуру через чат или «режим агента» (Composer).
*   **VS Code:** Классический редактор от Microsoft. В отличие от Cursor, он требует установки расширений (например, GitHub Copilot или Kimi Code) для работы с ИИ.
*   **Firebase Studio:** Новое облачное пространство от Google (на базе Code OSS), где ИИ-агенты помогают создавать приложения целиком: от прототипирования фронтенда по скриншотам до развертывания бэкенда.

## 2. Дизайн и визуализация

*   **Figma + ИИ-агенты:** Figma теперь поддерживает ИИ-агентов, которые могут не только «читать» макеты, но и самостоятельно рисовать их на канвасе.
*   **Связка Figma & Cursor:** С помощью протокола MCP (Model Context Protocol) можно подключить Figma напрямую к Cursor. Это позволяет ИИ-агенту в редакторе видеть ваши дизайны и писать код верстки, основываясь на макетах в реальном времени.

## 3. Модели и ИИ-агенты (Мозги)

*   **Kimi (от Moonshot AI):** Китайская нейросеть, известная огромным контекстным окном (до 2 млн знаков) и мощными способностями к программированию.
*   **Kimi Code:** Специализированный агент, который может работать в терминале или как расширение для VS Code, самостоятельно планируя и выполняя многоэтапные задачи.
*   **Kimi K2.5:** Последняя мультимодальная модель, на которой часто базируются современные ИИ-инструменты (например, Composer 2 в Cursor).
*   **DeepSeek (диписк):** Еще одна мощная китайская модель (DeepSeek-V3/R1), которая стала крайне популярной благодаря высокой производительности в кодинге при низкой цене API. Её часто подключают к Cursor или VS Code как основную «голову» для генерации кода.

## Итог: как это работает вместе

Вы рисуете дизайн в Figma, подключаете его к Cursor (или VS Code), а в качестве «двигателя» выбираете Kimi или DeepSeek. ИИ-агент (например, в режиме Composer) берет макет, анализирует ваш текущий код и самостоятельно вносит изменения сразу в несколько файлов, создавая готовое приложение.
