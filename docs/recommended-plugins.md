# Рекомендуемые плагины для WordPress

Для реализации всего функционала, представленного в статическом макете (новости, мероприятия, услуги, каталог, карта), стандартных возможностей WordPress недостаточно. Ниже приведен список рекомендуемых плагинов, которые помогут вам построить сайт с нужной структурой и возможностями.

### 1. Плагины для создания кастомных типов контента и полей

Чтобы отделить "Услуги", "Книги" и "Мероприятия" от обычных новостей, необходимо создать собственные типы записей.

*   **Custom Post Type UI (CPT UI)**
    *   **Зачем нужен:** Этот плагин позволяет легко, через простой графический интерфейс, создавать новые типы контента (например, "Услуги", "Книги", "События") без необходимости писать PHP-код.
    *   **Ссылка:** [https://wordpress.org/plugins/custom-post-type-ui/](https://wordpress.org/plugins/custom-post-type-ui/)

*   **Advanced Custom Fields (ACF)**
    *   **Зачем нужен:** Это один из самых важных плагинов. Он позволяет добавлять к вашим записям (книгам, услугам, событиям) любые дополнительные поля. Например, для книги это могут быть поля "Автор", "Год издания", "ISBN", а для услуги — "Цена" и "Срок выполнения".
    *   **Ссылка:** [https://wordpress.org/plugins/advanced-custom-fields/](https://wordpress.org/plugins/advanced-custom-fields/)

### 2. Плагин для раздела "Мероприятия"

*   **The Events Calendar**
    *   **Зачем нужен:** Мощный и популярный плагин для создания и управления мероприятиями. Он предоставляет удобный календарь, списки предстоящих событий, виджеты и всю необходимую логику для работы с датами, местами проведения и организаторами.
    *   **Ссылка:** [https://wordpress.org/plugins/the-events-calendar/](https://wordpress.org/plugins/the-events-calendar/)

### 3. Плагин для улучшенного поиска по каталогу

Стандартный поиск WordPress не ищет по содержимому кастомных полей, что критично для электронного каталога.

*   **Relevanssi**
    *   **Зачем нужен:** Этот плагин заменяет стандартный поиск WordPress на более интеллектуальный и позволяет индексировать и искать информацию в кастомных полях ACF (например, по автору или году издания книги).
    *   **Ссылка:** [https://wordpress.org/plugins/relevanssi/](https://wordpress.org/plugins/relevanssi/)

### 4. Плагин для интерактивной карты

*   **WP Go Maps (ранее WP Google Maps)**
    *   **Зачем нужен:** Позволяет легко создавать кастомные карты Google Maps с вашими метками, описаниями и маршрутами. Идеально подходит для визуализации расположения корпусов, отделов или филиалов на карте.
    *   **Ссылка:** [https://wordpress.org/plugins/wp-google-maps/](https://wordpress.org/plugins/wp-google-maps/)

### 5. Плагин для Headless CMS (Опционально)

Если вы планируете использовать WordPress только как систему управления контентом (бэкенд), а фронтенд разрабатывать на React/Next.js (как в исходных `.tsx` файлах), вам понадобится удобный способ получать данные.

*   **WPGraphQL**
    *   **Зачем нужен:** Предоставляет современный GraphQL API для вашего WordPress-сайта. Это позволяет вашему фронтенд-приложению запрашивать и получать только те данные, которые ему необходимы, что очень эффективно.
    *   **Ссылка:** [https://wordpress.org/plugins/wp-graphql/](https://wordpress.org/plugins/wp-graphql/)

## Итог

Установка и настройка этих плагинов даст вам прочную основу для реализации всего задуманного функционала и создания гибкого, легко управляемого сайта на WordPress.
