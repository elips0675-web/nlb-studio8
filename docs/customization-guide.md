
# Руководство по кастомизации проекта

## 1. Важное замечание: Этот проект — не Bitrix

Прежде всего, важно понять: **этот проект создан с использованием React, а не 1С-Битрикс**.

Это два совершенно разных подхода к созданию сайтов:
*   **Этот проект (React):** Современное JavaScript-приложение. Вся структура, страницы и логика описываются в виде **React-компонентов** (`.tsx` файлы). Он не имеет ничего общего с архитектурой, шаблонами или API Bitrix.
*   **1С-Битрикс:** Это система управления контентом (CMS) со своей серверной логикой на PHP, базой данных и собственной системой шаблонов.

**Вывод:** Вы не можете "установить этот шаблон на Bitrix" или "кастомизировать Bitrix с помощью этого сайта". Это технически несовместимые системы.

Если вам нравится дизайн этого проекта и вы хотите применить его к своему сайту на Bitrix, вам потребуется **вручную воссоздать** весь HTML, CSS и логику с нуля внутри вашей Bitrix-системы. Это сложная задача для опытного Bitrix-разработчика.

---

## 2. Как на самом деле устроен этот проект

*   **Фреймворк:** React (вероятнее всего, с использованием Next.js, судя по структуре).
*   **Стилизация:** Tailwind CSS. Все стили задаются с помощью классов прямо в HTML-структуре (например, `text-2xl font-bold`).
*   **"Шаблоны" и "Страницы":** Это просто React-компоненты, расположенные в `src/app/page-templates/`. Каждый `.tsx` файл в этой папке — это, по сути, шаблон или полноценная страница.

## 3. Руководство по кастомизации

### 3.1. Изменение текстов и изображений

Большая часть контента (тексты, заголовки, цены, ссылки на изображения) "зашита" прямо в коде компонентов. Чтобы их изменить, вам нужно найти соответствующий файл и отредактировать его.

**Пример 1: Редактирование новости**

1.  **Найдите файл:** Страница с новостями, скорее всего, находится в `src/app/page-templates/all-news-page.tsx`.
2.  **Найдите блок данных:** Внутри файла вы увидите массив `newsItems`, где каждая новость — это объект.
    ```javascript
    const newsItems = [
      {
        id: 1,
        title: 'Летний читальный зал открыт!', // <-- Ваш заголовок
        date: '15 июня 2024', // <-- Ваша дата
        description: 'Наслаждайтесь книгами на свежем воздухе...', // <-- Ваше описание
        image: 'https://images.unsplash.com/...', // <-- Ваша ссылка на картинку
      },
      // ...другие новости
    ];
    ```
3.  **Отредактируйте:** Просто измените текст в кавычках на нужный вам.

**Пример 2: Редактирование услуги печати**

1.  **Найдите файл:** Детальная информация об услугах находится в `src/app/page-templates/printing-service-detail.tsx` и `src/app/data/printing-services.ts`.
2.  **Найдите блок данных:** Внутри `printing-service-detail.tsx` есть большой объект `printingServiceDetails`.
    ```javascript
    const printingServiceDetails = {
      't-shirts': { // <-- ID услуги
        id: 't-shirts',
        icon: Shirt,
        title: 'Печать на футболках', // <-- Название
        subtitle: 'Футболки и свитшоты с вашим дизайном', // <-- Подзаголовок
        fullDescription: 'Предлагаем прямую цифровую печать (DTG)...', // <-- Полное описание
        images: [
            'https://images.unsplash.com/...', // <-- Ссылка на картинку
        ],
        // ...
      }
    };
    ```
3.  **Отредактируйте:** Измените текстовые поля и ссылки на изображения.

### 3.2. Изменение стилей (цвета, размеры, отступы)

Стили задаются классами Tailwind CSS.

*   `bg-blue-500`: задает синий цвет фона.
*   `text-2xl`: делает текст большим (размер 2xl).
*   `font-bold`: делает шрифт жирным.
*   `p-4`: добавляет внутренний отступ (padding) со всех сторон.
*   `m-8`: добавляет внешний отступ (margin) со всех сторон.

Чтобы изменить внешний вид, найдите нужный HTML-элемент в `.tsx` файле и измените его классы.

**Пример: Изменение цвета кнопки**

Найдите кнопку в коде:
```jsx
<button className="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
  Нажми меня
</button>
```

Чтобы сделать ее зеленой, замените `bg-blue-500` на `bg-green-500`:
```jsx
<button className="bg-green-500 hover:bg-green-700 text-white font-bold py-2 px-4 rounded">
  Нажми меня
</button>
```

### 3.3. Добавление новой страницы (например, новой услуги)

1.  **Скопируйте шаблон:** Создайте копию существующего файла шаблона, например, `src/app/page-templates/page-templates/business-cards-template.tsx` и переименуйте его, например, в `my-new-service-template.tsx`.
2.  **Отредактируйте контент:** Откройте новый файл и измените в нем весь контент (заголовки, тексты, цены, изображения) на тот, который вам нужен для новой услуги.
3.  **Добавьте услугу в каталог:** Откройте файл `src/app/data/printing-services.ts`. Добавьте в массив `catalogServices` новый объект, описывающий вашу услугу. Убедитесь, что `id` в этом объекте уникален.
    ```javascript
    export const catalogServices = [
      // ...существующие услуги
      {
          id: 'my-new-service', // <-- Уникальный ID
          icon: FileText, // Выберите иконку
          title: 'Моя новая услуга',
          description: 'Краткое описание моей новой услуги.',
          image: 'https://.../new_image.jpg',
          prices: [
            { name: 'Цена 1', price: '100₽' },
          ],
          color: 'from-yellow-500 to-orange-500',
      },
    ];
    ```
4.  **Свяжите каталог с детальной страницей:** Откройте `src/app/page-templates/printing-service-detail.tsx` и добавьте в объект `printingServiceDetails` описание для вашей новой услуги, используя тот же `id`.
    ```javascript
    const printingServiceDetails = {
        // ...существующие услуги
        'my-new-service': {
            id: 'my-new-service',
            title: 'Моя новая услуга',
            // ... все остальные поля, как у других услуг
        }
    }
    ```

После этого новая услуга должна появиться в общем каталоге, и по клику на нее будет открываться страница с подробным описанием.
